1         ВВЕДЕНИЕ
1.1        Описание проекта 
Разрабатываемая база данных предназначена для учета ремонтных работ, выполненных на вагонах, фиксации данных о договорах на ремонт, а также расчета оплаты сдельной работы для исполнителей.
2        ОСНОВАНИЕ ДЛЯ РАЗРАБОТКИ 
2.1         Основание для проведения разработки 
Основанием для данной работы служит договор №1 от 06.02.2025г.


2.2        Наименование и условное обозначение темы разработки 
Разработка системы автоматизации организационных процессов при ремонте вагонного оборудования “Система управления ремонта вагонного оборудования”.
3        НАЗНАЧЕНИЕ РАЗРАБОТКИ 
Назначение разработки:
Упростить ведение отчетности и ускорить процесс администрирования ремонтов вагонов
3.1.  Основные сущности базы данных
3.1.1 Вагон
Каждый вагон должен содержать следующую информацию:
* Номер вагона (например, 024-06064)
* Собственник вагона (например: ДОСС, ФПК, Гранд Экспресс и др
* Подразделение приписки/управления (например, ЛВЧ/ЛВЧД/)
* Дата проведения КР/КВР/ДР
3.1.2 Договор
Для каждого договора необходимо учитывать:
* Номер договора (например, № 2024.288648)
* Дата договора (например, 23 декабря 2024 г.)
* Список услуг, предусмотренных данным договором со стоимостью без НДС и с НДС
3.1.3 Услуга
Каждая услуга должна содержать:
* Наименование услуги
* К какому договору принадлежит услуга (просто привязка к договору для правильного выбора стоимости)
* Стоимость за единицу услуги без учета НДС и с НДС
* Возможность загрузки списка услуг и стоимости из EXEL-файла (изменения списка в программе)
3.1.4 Выполненные работы
Оператор вручную - (выбирает) информацию о выполненных работах:
выбирая номер договора, попасть на вкладку выбора услуг, далее набор оказанных услуг
* Исполнитель (ФИО работника) (занести 1 раз и выбирать потом)
* Дата выполнения услуги (интервал времени, например с 11до17)
* Кто подписывает данную работу 
* Услуга
* Номер договора
* Вагон, на котором выполнена услуга
3.1.5 Оплата работников
Оплата работника рассчитывается по сдельной системе:
* ФИО исполнителя
* выборка по интервалам дат
* Перечень выполненных работ
* Стоимость выполненных работ
* Общая сумма сдельной платы


3.2.  Функциональные задачи программы:
3.2.1 Учет выполненных работ
* Оператор - (выбирает) - информацию согласно пункту 3.1.4.
3.2.2 Импорт списка услуг
* При загрузке EXEL файла система  должна автоматически  подгружать их в  пункт 3.1.2
* Возможность вручную добивать данные 

3.2.3  Формирование отчетных документов
3.2.3.1 АКТ выполненных работ
Должен автоматически формироваться файл содержащий таблицу со следующей структурой: (форма акта согласно шаблона)
   * Буква исполнителя
   * Дата выполнения работ
   * Порядковый № акта
   * Номер вагона
   * Номер договора
   * Подразделение приписки вагона
   * Перечень выполненных услуг и их стоимость
   * Итоговая стоимость
   * Объем ТО (250, 500 или 1000)
   * ФИО исполнителя услуг
3.2.3.2 Заявка на работы
Формируется документ по шаблону на основании следующей информации:
   * Дата заявки
   * Номер договора
   * Номер вагона
   * Интервал выполнения работ
   * Конечная дата выполнения работ (совпадает с датой акта)
        3.2.3.3 Выписка по договорам
Для каждого договора должна формироваться таблица со следующими столбцами:
   * Наименование услуги
   * Стоимость за единицу услуги (без НДС и с НДС)
   * Кол-во 
   * Номера Вагонов
   * Итого (Без НДС и с НДС)


4        ПОЛЬЗОВАТЕЛЬСКИЙ ПУТЬ
4.1        Древо пользовательского пути:
Добавление данных
   * Добавление вагона
   * Добавление договора
   * Добавление услуги (вручную / импорт из Word/EXEL)
Выполнение работ
   * Запись выполненной работы
   * Указание исполнителя
   * Привязка к договору и вагону
Отчетность и документы
   * Формирование акта
   * Формирование выписки по договору
   * Создание заявки на работы
Финансы и расчеты
   * Расчет сдельной оплаты работников
   * Анализ выполненных услуг (3.2.3.3 )
Экспорт и загрузка данных
   * Импорт услуг из EXEL
   * Экспорт отчетов (Excel, PDF)
